Data Capture — How Conversations Are Persisted
recordConversation() is called twice per step — once after Claude responds, once after tool results are appended.
1
Claude called with messages[]Growing array contains all prior turns. Claude returns content blocks.
2
Assistant message pushedmessages.push({ role: 'assistant', content: response.content })Content: TextBlock + ToolUseBlock objects.
3
recordConversation() — FIRST CALLPersists after Claude's response, before tool results. stepNumber = currentStep + 1.
4
MCP tools executedEach tool_use block dispatched to Playwright MCP. Results truncated to 10,000 chars.
5
Tool results pushed as user messagemessages.push({ role: 'user', content: toolResults })
6
recordConversation() — SECOND CALLComplete turn persisted. This is the authoritative snapshot.

Modal Training — Under the Hood
Two Modal functions: kickoff (web, no GPU, 150s timeout) spawns train (A10G GPU, 6h timeout). Avoids the 150s hard limit.

QLoRA

Quantized Low-Rank Adaptation

Quantized base (4-bit, frozen) + trainable LoRA adapters (16-bit). ~3.5GB base + ~100MB adapters. Fits 24GB VRAM.

LORA_R = 16          # rank of adapter matrices
LORA_ALPHA = 16      # scaling (alpha/r = 1.0)
LEARNING_RATE = 2e-4
BATCH_SIZE = 2 x GRAD_ACCUM = 4  # effective batch 8
NUM_EPOCHS = 3
TARGETS = ['q_proj','k_proj','v_proj','o_proj','gate_proj','up_proj','down_proj']

Common Questions
Why does the first step get cold-start SSE but not later steps?+
Timer only set when stepNumber === 0. Modal vLLM takes ~2min on cold start. Warm-up ping during browser creation overlaps most of this.
Why is temperature 0.0 for fine-tuned but 1.0 for Claude?+
Temp=0 makes eval reproducible. Claude uses 1.0 for diverse training trajectories.
Why minSteps=1 for one-click but 3 for CLI?+
You've already hand-selected games in the UI. Aggressive filtering would silently drop sessions you specifically picked.
NEXT_PUBLIC_CONVEX_SITE_URL vs NEXT_PUBLIC_CONVEX_URL?+
CONVEX_URL (.convex.cloud) = WebSocket client SDK. CONVEX_SITE_URL (.convex.site) = HTTP endpoint for external services like Modal.
What if I select 5 games but 2 are fine-tuned model games?+
The 2 are silently dropped (0 tool_use blocks). Pipeline continues with the 3 valid Claude Sonnet games.
Why Modal volumes instead of downloading each time?+
Base model is ~6GB. Volume persists at /model_cache. First run downloads; subsequent runs load from disk instantly.
How does modal.parameter() serve multiple experiments?+
Each unique experiment_name gets its own container pool. 10 experiments = 10 isolated vLLM processes, one modal deploy.
save_pretrained() vs save_pretrained_merged()?+
save_pretrained() = LoRA adapter only (~100MB). save_pretrained_merged("merged_16bit") = full model with adapter baked in (~6GB). vLLM needs the merged form.
Why does convertToolResults() look backward through messages?+
tool_result only has tool_use_id, not the tool name. findToolName() scans backward to find the matching tool_use block.
What if MODAL_TRAIN_ENDPOINT is missing on Vercel?+
